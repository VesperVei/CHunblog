# PWN利用技术分类

## 技术体系总览
PWN利用技术是将理论漏洞转化为实际攻击能力的关键环节，涵盖了从基础控制流劫持到高级代码复用的完整技术链条。本笔记作为利用技术的总索引，旨在系统化地组织和呈现各类利用方法，为实际漏洞利用提供清晰的技术路径选择。
## 核心利用技术详解
从学习路径上看，可以先建立最基础的控制流劫持直觉，再逐步进入 ==代码复用==、==调用构造== 与 ==复杂环境绕过==。

### 基础控制流劫持技术
基础控制流劫持技术是理解 Pwn 利用链的起点，主要帮助建立“覆盖返回地址后，程序会跳到哪里”的基本认知。

[[ret2text]] 技术通过覆盖返回地址，使程序跳转到代码段中已有的敏感函数或代码片段，例如 `system`、`win`、`backdoor` 等。这种技术简单直接，通常用于题目本身已经暴露出可利用目标函数的场景，是新手理解控制流劫持最自然的切入点。

[[ret2shellcode]] 技术要求将 shellcode 注入到可执行内存区域，并通过覆盖返回地址跳转到 shellcode 起始位置执行。它代表了最经典的“注入并执行”利用思路，但前提是**目标内存区域必须具备执行权限**。在现代 NX 普及的环境下，这种方式的适用范围受到限制，不过在关闭 NX、存在 RWX 段或特定平台场景下仍然具有代表性。

[[ret2libc]] 技术是现代利用中最重要的基础模型之一。它并不依赖注入新的可执行代码，而是直接复用 libc 中现成的函数，例如 `system`、`execve` 等，借此完成命令执行或进一步控制程序流程。它有效绕过了 NX 对 shellcode 的限制，因此在实际题目中比传统 [[ret2shellcode]] 更常见，也更贴近现代栈溢出利用思路。

### 中级控制流劫持技术

中级控制流劫持技术的重点不再只是“跳到某个现成地址”，而是开始进入 ==代码复用== 与 ==调用构造== 的阶段。学习者需要理解调用约定、寄存器控制、栈布局，以及在受限条件下如何组织完整的利用链。

[[ROP]]（Return-Oriented Programming）是现代控制流劫持中的核心技术。它通过在程序本体或其依赖库中寻找以 `ret` 指令结尾的代码片段（Gadgets），并将这些片段的地址按顺序布置到栈上，使程序在一次次返回中连续执行攻击者预期的逻辑。  
这种技术不依赖注入新的可执行代码，而是复用已有指令片段组合功能，因此是绕过 NX 保护的关键手段。和基础跳转技术相比，[[ROP]] 更强调寄存器控制、参数传递、栈对齐以及 Gadget 组织能力，是进入进阶利用的分水岭。

[[ret2syscall]] 技术是通过控制处理器寄存器（如 `eax`、`rax`、`rdi`、`rsi`、`rdx` 等）来直接触发系统调用。它通常结合 [[ROP]] 链使用，通过 Gadget 完成寄存器布置后，跳转到 `syscall` 或 `int 0x80` 指令处执行。  
这种方式不依赖 libc 中具体函数的地址，只要能控制系统调用号和参数，并找到可用的系统调用入口，就能完成利用。因此它常用于 libc 条件受限、静态链接或更偏底层控制的场景。

[[GOT表劫持]]（Global Offset Table Overwrite）利用的是 ELF 动态链接中的延迟绑定机制。程序在调用外部函数时，会通过 PLT 和 GOT 完成跳转；如果攻击者能够通过漏洞修改 GOT 表项内容，就可以把原本指向某个函数的地址替换为 `system`、`one_gadget` 或其他目标地址。这样，当程序后续再次调用该函数时，控制流就会跳转到攻击者指定的位置。  
这种技术对利用环境依赖较强，尤其与 `RELRO` 配置密切相关，因此常见于动态链接机制相关的进阶题目中。

[[ret2csu]] 是 x86_64 架构下非常实用的寄存器控制技巧，主要用于常规 Gadget 匮乏、难以直接控制 `rdx` 等关键寄存器的场景。它利用几乎所有 ELF 程序中都存在的 `__libc_csu_init` 函数末尾的两段固定代码片段，通过构造调用流程来稳定控制 `rdi`、`rsi`、`rdx` 等参数寄存器，并完成对任意函数地址的调用。  
在很多 Gadget 不够理想的题目里，[[ret2csu]] 往往是补齐利用链的重要手段。

[[栈迁移]]（Stack Pivoting）是指通过修改 `esp` 或 `rsp`，把程序后续使用的栈切换到攻击者事先准备好的伪造栈区域，例如 `.bss`、堆区或其他可控内存位置。  
它最常见的应用场景是：原始漏洞可覆盖的栈空间太小，不足以直接放下完整的 [[ROP]] 链。这时就需要先借助 `leave; ret`、`xchg rsp, reg` 等 Gadget 将栈迁移出去，再在新的可控区域上继续执行后续利用链。

[[ret2dlresolve]]（Return-to-dl-resolve）是一种在没有 libc 基址泄露、又无法直接得到目标函数地址时非常有价值的技术。它通过在栈上或其他可控区域中伪造重定位表项、符号表项和字符串表内容，诱导动态链接器在运行时解析并调用攻击者指定的函数，例如 `system`。  
这种技术对 ELF 动态链接流程的理解要求较高，涉及 `STRTAB`、`SYMTAB`、`JMPREL` 等结构，因此通常出现在对二进制加载机制理解要求更深的题目中。

### 高级控制流劫持技术

高级控制流劫持技术通常不再局限于传统的返回地址覆盖与常规代码复用，而是进一步利用 ==内核信号机制==、==黑盒探测能力== 或 ==语言运行时对象模型== 来完成控制流操纵。这类技术往往出现在保护更严、环境更复杂的利用场景中。

[[SROP]]（Sigreturn Oriented Programming）是一种通过模拟 Linux 信号处理机制来控制寄存器状态的高级利用技术。它的核心思路是伪造一个信号上下文结构，并触发 `rt_sigreturn` 系统调用，使内核从该伪造结构中一次性恢复所有通用寄存器的值。  
与常规 [[ROP]] 相比，[[SROP]] 的优势在于：当 Gadget 极度匮乏、无法通过普通 `pop` 链完成关键寄存器控制时，它可以直接实现接近“全寄存器接管”的效果。因此它在受限环境下具有很强的实战价值。

[[BROP]]（Blind ROP）是一种典型的黑盒利用技术，适用于攻击者拿不到目标二进制文件、不了解源码、也不知道精确内存布局的远程场景。它的核心在于：结合目标程序的崩溃与非崩溃反馈，通过不断探测来定位 stop gadget、可调用 gadget 和泄露原语，最终逐步拼出完整的利用链。  
这种技术的难点不只在 Gadget 本身，而在于整个探测过程必须建立在极少信息和反复试探的前提下，因此对利用建模能力和远程交互稳定性要求都很高。

[[COOP]]（Counterfeit Object-Oriented Programming）是面向 C++ 程序的一种高级控制流劫持方法。它并不直接覆盖返回地址，而是通过伪造对象及其虚函数表指针，引导程序沿着看似合法的虚函数调用路径执行攻击者设计的一系列操作。  
这种技术的特殊之处在于：它利用的是面向对象程序内部本来就合法存在的控制流，而不是传统的“跳到不该去的地址”。因此它常被用于对抗现代基于控制流完整性（CFI）的防护机制，属于更偏语言运行时与高级防护绕过的专题技术。

对于入门 Pwn 的学习者来说，这一整套技术的理解路径可以概括为：  
先通过 [[ret2text]]、[[ret2shellcode]]、[[ret2libc]] 建立最基本的控制流劫持直觉，再通过 [[ROP]]、[[ret2syscall]]、[[GOT表劫持]]、[[ret2csu]]、[[栈迁移]]、[[ret2dlresolve]] 学会在现代保护下组织调用与扩展利用链，最后再进入 [[SROP]]、[[BROP]]、[[COOP]] 这些更依赖运行时机制与复杂环境建模的高级专题。
### 堆利用技术体系

堆利用技术针对堆内存管理漏洞，具有高度的复杂性和平台依赖性。

**[[堆布局操控]]** 技术通过精心设计的内存分配和释放序列，影响堆的宏观布局结构，为后续利用创造有利条件。这种技术需要深入理解特定堆管理器的分配策略和内存组织结构。

**[[元数据攻击]]** 技术直接针对堆块的管理元数据进行操作，通过破坏或篡改元数据信息实现任意地址读写能力。这类技术是堆利用的核心，包括unlink攻击、overlapping等经典手法。

### 现代防护绕过技术

现代系统部署了多层次防护机制，绕过这些防护成为利用技术的重要组成部分。

**信息泄露技术** 旨在获取关键地址信息，以绕过ASLR和PIE等随机化保护。常见的泄露方式包括格式化字符串漏洞、堆地址泄露、栈地址泄露等。

**部分覆盖技术** 在有限的内存写入能力下，通过精准的部分字节覆盖实现关键数据修改。这种技术常用于绕过Stack Canary或实现精确的地址篡改。

### 高级组合利用技术

实际攻击中往往需要组合多种技术形成完整的利用链条。

**多阶段利用** 将复杂的攻击过程分解为多个阶段，通过分步执行降低单次利用的难度和复杂度。这种技术适用于限制较多或稳定性要求高的场景。

**条件竞争利用** 利用程序中的时序相关漏洞，通过精确的竞争条件触发实现内存破坏或权限提升。这类技术要求对程序执行时序有深入理解。

## 技术选择与演进趋势

### 技术适用性分析

不同利用技术适用于不同的漏洞类型和环境条件。基础控制流劫持技术适用于栈溢出等传统漏洞，代码复用技术更适合现代防护环境，堆利用技术专门针对堆相关漏洞，而防护绕过技术则是各类利用的基础支撑。

### 技术演进路径

利用技术的发展呈现出明显的阶段性特征。早期以直接控制流劫持为主，中期转向代码复用攻击，现代则更加注重防护绕过和组合利用。未来技术发展将更加关注可靠性、通用性和自动化程度。

## 学习路径建议

建议按照基础到高级的顺序系统学习利用技术：

1. 从[[ret2text]]和[[ret2shellcode]]开始，建立控制流劫持的直观理解
2. 掌握[[ret2libc]]技术，理解现代利用的基本范式
3. 深入学习[[ROP]]，掌握代码复用的核心方法
4. 研究堆利用和高级组合技术，提升复杂环境下的利用能力
