# Theme And Runtime

This document explains the browser-side behavior and build integrations that affect the rendered site.

## Site Theme

The site supports three visual theme modes:

- `light`
- `dark`
- `system`

Theme state is stored in `localStorage` under the `theme` key. The active mode is applied to the root element with `data-theme`.

- `data-theme="light"` forces light mode.
- `data-theme="dark"` forces dark mode.
- No `data-theme` means system mode.

The runtime logic lives in `src/utils/theme-script.ts` and is injected by `src/components/theme/ThemeScript.astro`, which is used by the header.

## Theme Variables

Theme CSS variables are defined in `src/sass/_variables.scss`.

The `theme-variables` mixin emits:

- default light variables under `:root`
- forced dark variables under `[data-theme="dark"]`
- system dark variables under `@media (prefers-color-scheme: dark)` when `data-theme` is not `light`

Most visual styling uses CSS variables such as `--bg-color`, `--fg-color`, `--accent-color`, `--glass-bg`, `--shadow`, and `--rounded-corner`.

## Accent Color

Accent color behavior is configured in `siteConfig.theme_color` in `src/config.ts`.

Supported modes:

- `hue`: user chooses hue; script derives a lightness value based on active light or dark mode.
- `hsl`: user chooses full HSL values.
- `fixed`: static color with no UI controls.

The client script stores custom values in:

- `custom-theme-hue`
- `custom-theme-hsl`

When the active theme changes, the script reapplies the accent color so derived values stay readable.

## Expressive Code

Expressive Code is configured in `astro.config.mjs`.

Current behavior:

- Uses `github-light` and `github-dark`.
- Uses `useDarkModeMediaQuery: true` so system mode follows `prefers-color-scheme`.
- Uses `themeCssSelector: (theme) => \`[data-theme='${theme.type}']\`` so forced site theme switches also switch code block colors.
- Uses `pluginLineNumbers()` for line numbers.
- Enables wrapped code blocks by default.
- Disables line numbers for `shellsession`.
- Uses project CSS variables for code font, border radius, frame shadow, and active tab accent color.

This means code blocks follow the same light, dark, and system behavior as the rest of the site.

## Mermaid

Mermaid support is implemented with:

- `src/utils/remark-mermaid.js`
- `src/components/common/MermaidScript.astro`

The Mermaid script currently initializes Mermaid with the `neutral` theme.

## Math

Math support is configured in `astro.config.mjs` with:

- `remark-math`
- `rehype-katex`

KaTeX CSS is imported in `src/components/shell/Head.astro`.

## Search

Search uses Pagefind.

- The Astro integration `astro-pagefind` is registered in `astro.config.mjs`.
- The production build command is `astro build && pagefind --site dist`.
- Search UI components live in `src/components/search/`.
- `siteConfig.search.enable` controls whether search should be available.

Build output creates a Pagefind index in `dist/pagefind`.

## RSS And Sitemap

RSS is controlled by `siteConfig.rss.enable`.

RSS routes:

- `src/pages/rss.xml.js`
- `src/pages/[lang]/rss.xml.js`

Sitemap is generated by `@astrojs/sitemap`, configured in `astro.config.mjs`.

## Comments

Comments use either Giscus or Waline when `siteConfig.comments` is configured in `src/config.ts`.

- `src/components/blog/Comments.astro` selects the configured provider and renders either `GiscusComments.astro` or `WalineComments.astro`.
- Giscus requires repo and category metadata and syncs its iframe theme with the active site theme.
- Waline requires `serverURL` and initializes the official `@waline/client` browser bundle with the current page path.
- `src/layouts/BlogPost.astro` places the comments block below the article body and above previous/next post navigation.
- Both providers react to `data-theme` and `prefers-color-scheme` so comment UI stays aligned with site light and dark mode.

### `siteConfig.comments`

`comments` is a discriminated union keyed by `provider`.

Shared shape:

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `provider` | `"giscus" \| "waline"` | yes | Selects the comment provider |

## Build Notes

Main build command:

```bash
pnpm build
```

Known warning observed during build:

- `src/content/demo.mdx` contains a code block language value `scss,`.
- Expressive Code does not know `scss,`, so it falls back to text.
- Fix by changing the code fence language to `scss`.
